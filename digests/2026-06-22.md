## PostgreSQL

- **[In-core DDL reconstruction starts to take shape](https://github.com/postgres/postgres/commit/2af70e9)** — the week's busiest patch thread ("Add pg_get_table_ddl()", 14 messages) and a cluster of commits point at SQL-callable DDL reconstruction — the thing you currently shell out to `pg_dump` for. Tom Lane [fixed the variadic declarations of the `pg_get_*_ddl()` functions](https://github.com/postgres/postgres/commit/2af70e9), Andrew Dunstan [switched them to named boolean parameters](https://github.com/postgres/postgres/commit/d6ed87d), and the "Get Object DDL Functions" docs table was committed at the window's edge. *(Tom Lane, Andrew Dunstan · postgres git)* `[committed]`
- **[Inherited CHECK constraints can no longer be silently weakened](https://github.com/postgres/postgres/commit/0cd17fd)** — a child table could relax a CHECK it inherited, so the parent's advertised guarantee no longer held across the hierarchy; the matching CF entry ("CHECK constraint enforceability recursion") was committed Jun 28. Paired with doc clarifications for `ALTER CONSTRAINT` and inherited-constraint behavior. *(Andrew Dunstan)* `[committed]`
- **[COPY TO (FORMAT json) respects an explicit column list's order](https://github.com/postgres/postgres/commit/effb923)** — `COPY t (b, a) TO … (FORMAT json)` now emits keys in the requested order, matching text/CSV. *(Andrew Dunstan)* `[committed]`
- **[Temporal `FOR PORTION OF` gets two beta fixes](https://github.com/postgres/postgres/commit/a272a58)** — Peter Eisentraut [moved the volatility check on the SQL:2011 period bounds into the planner](https://github.com/postgres/postgres/commit/a272a58) and [rejected child-partition foreign tables under `FOR PORTION OF`](https://github.com/postgres/postgres/commit/a40fdf6), hardening the new temporal feature before GA. *(Peter Eisentraut)* `[committed]`
- **[`MERGE`/`SPLIT PARTITION` now honors `default_tablespace`](https://github.com/postgres/postgres/commit/cdae794)** — reshaping a partitioned table no longer quietly lands data on the wrong tablespace. *(Alexander Korotkov)* `[committed]`
- **[A UUID work surge](https://github.com/postgres/postgres/commit/6468f7a)** — the `uuid`→`bytea` cast was [marked leakproof](https://github.com/postgres/postgres/commit/6468f7a) (so quals using it can sink below RLS/security-barrier views), and the CommitFest filled with UUID patches: SIMD-accelerated UUID parsing, `MIN`/`MAX` aggregates for `uuid`, `UUIDv6` support in `uuid_extract_timestamp()`, and a fix for `uuidv7` wrongly accepting dates before 1970. *(Masahiko Sawada + CF)* `[committed + patches]`
- **[`pg_stat_io` stops misattributing autovacuum-launcher extends](https://github.com/postgres/postgres/commit/4abf411)** — relation extends by the autovacuum *launcher* were counted like a worker's, skewing per-backend-type I/O accounting. *(Melanie Plageman)* `[committed]`
- **Beta-hardening grab-bag** — [out-of-bounds read fixed in the autoprewarm worker](https://github.com/postgres/postgres/commit/dac3660) *(Tomas Vondra)*; the [online data-checksums worker got a four-commit correctness pass](https://github.com/postgres/postgres/commit/a4f02ca) *(Heikki Linnakangas)*; [null-pointer crash fixed in the ECPG compiler](https://github.com/postgres/postgres/commit/7f5e0b2) *(Tom Lane)*; [`\crosstabview` now honors `\pset` boolean/null display](https://github.com/postgres/postgres/commit/4df5fe3) *(Álvaro Herrera)*; and [collation lookup is avoided for `"char"` statistics](https://github.com/postgres/postgres/commit/b574fec) *(Tom Lane)*. `[committed]`

### Reading from Planet PostgreSQL
- **[Some more thoughts on random_page_cost](https://postgr.es/p/9na)** — Tomas Vondra revisits whether the SSD-era default (4.0) still makes sense, following up on his pre-recorded POSETTE talk; the planner's random-vs-sequential cost ratio is increasingly wrong on NVMe. *(Tomas Vondra)*
- **[Same rows, different SUM](https://postgr.es/p/9nx)** — Radim Marek sums a 5-million-row `float8` column three times on an untouched table and gets three different answers; a clean tour of why parallel aggregation + floating-point non-associativity makes `SUM(double precision)` non-deterministic. *(Radim Marek · boringsql)*
- **[Stop Punishing Your Postgres for a Crash That Won't Happen](https://postgr.es/p/9nu)** — Lætitia Avrot dismantles the widespread fear that a longer `checkpoint_timeout` means a longer crash recovery, and why setting it to 1–5 minutes hurts steady-state performance. *(Lætitia Avrot)*
- **[Heterogeneous Graphs in SQL/PGQ on PostgreSQL 19](https://postgr.es/p/9nk)** — Hans-Jürgen Schönig exercises PG19's in-core SQL/PGQ property-graph querying (`GRAPH_TABLE`) over a graph with multiple vertex/edge labels. *(Hans-Jürgen Schönig · Cybertec)*
- **[pg_stats: How Postgres Internal Stats Work](https://postgr.es/p/9mG)** — Richard Yen on the statistics model behind seq-scan-vs-index-scan planner decisions. *(Richard Yen)*
- **[All Your GUCs in a Row](https://postgr.es/p/9nv)** — Christophe Pettus ran a near-daily series of short planner-GUC deep-dives this week (`enable_hashjoin`, `enable_hashagg`, `enable_gathermerge`, `enable_async_append`, `effective_io_concurrency`, distinct/group-by reordering). *(Christophe Pettus · thebuild.com)*

## PostgreSQL mailing lists (what's being designed)

Scanned the full Jun 22–28 [pgsql-hackers archive](https://www.postgresql.org/list/pgsql-hackers/2026-06/). Most active design threads:

- **Conflict log history table for Logical Replication** — by a wide margin the week's busiest design thread (~35 messages, 5–6 authors across the window): a PG20-targeted feature letting a subscription persist apply-time conflicts to a system-managed table rather than only the server log. *(Amit Kapila, Peter Smith, Nisha Moond, Dilip Kumar)*
- **The PostgreSQL C Dialect** — codifying which C features the tree relies on (see AI section above).
- **New SQL surface** — proposals for a `PRODUCT()` aggregate, `MIN`/`MAX` for `uuid`, and an `ON EMPTY` clause for aggregate/window functions; plus the long-running **Row pattern recognition** (SQL:2016 RPR) thread.
- **[Reduce reorder buffer spill from O(N²) to O(N) during vacuum storms](https://commitfest.postgresql.org/59/activity/)** — a logical-decoding performance patch landed in the CommitFest this week, attacking quadratic spill behavior when many concurrent transactions interleave. *(new CF record)*
