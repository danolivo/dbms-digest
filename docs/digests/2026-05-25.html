DBMS Weekly — 2026-05-31 (week of 2026-05-25–2026-05-31)

PostgreSQL

  • How soon is now()? A frozen-timestamp retry-loop bug — Oskar Dudycz walks through a real distributed-locking bug: now() returns the transaction start time, so a retry loop sharing one transaction kept re-evaluating last_updated < now() - timeout against a constant and never made progress; fix is clock_timestamp(). Includes the testing-seam lesson on why unit + e2e tests both missed it. (event-driven.io)
  • 5 PostgreSQL locking behaviours that trip people up — Shinya Kato (PostgreSQL engineer, NTT) on counterintuitive production-outage locks: FIFO lock-queue chained blocking behind a waiting ACCESS EXCLUSIVE, invisible FK FOR KEY SHARE deadlocks, unique-check INSERT deadlocks, the wraparound-prevention autovacuum that won't be cancelled, and VACUUM's hidden truncate-phase ACCESS EXCLUSIVE (and how it forcibly cancels standby SELECTs). (dev.to)
  • PostgreSQL Anonymizer 3.1 — Local Differential Privacy + critical CVE — Dalibo adds a formally-bounded LDP masking mechanism (Generalized Randomized Response, epsilon-tunable) for survey/categorical data, and fixes CVE-2026-9617, a privilege-escalation-to-superuser flaw (high risk on PG 14 and clusters upgraded from ≤14). Upgrade urged; SQL workaround posted. (Dalibo · postgresql.org)

PostgreSQL conferences (PGConf.dev 2026 trip reports)

  • PGConf.dev 2026: HA cluster-topology, wait-events, collation — Jeremy Schneider's trip summary surfaces real design directions discussed in Vancouver: Postgres lacks a first-class "cluster/topology" concept (a pg_nodes-style view, promotion-candidate discovery on standbys); Sami Imseih's pg_stat_statements LWLock/query-text-file concurrency patches; per-wait-event counters; and the unsolved ICU-upgrade-invalidates-indexes collation problem (with Oracle's pragmatic per-operator default as a reference). (ardentperf.com)
  • PGConf.dev 2026 reflections — SQL/PGQ, multithreading, resize-shared-buffers — Ashutosh Bapat reports on the graph-DB developer meeting (SQL/PGQ + AGE), the logical-replication working group (DDL replication and gaps before LR is usable for high-volume OLAP), the multithreading effort, and a quick design agreement with Databricks engineers (Heikki et al.) on resizing shared buffers without restart, built on the PG 19 shared-memory rework. (ashutoshpg.blogspot.com)

Releases

  • Percona Operator for PostgreSQL 3.0.0 — completes the hard fork from Crunchy PGO. (Percona)
    • inherited CRDs renamed under upstream.pgv2.percona.com, so the Crunchy and Percona operators can now coexist in the same cluster/namespace (no CRD/webhook/finalizer collision); one-time auto-migration on upgrade
    • fixed OpenShift OLM namespace scoping (watch list now honours the OperatorGroup)
    • major-version upgrades now use the official percona-distribution-postgresql-upgrade image so pg_upgrade runs the same signed binaries as production; note: pgaudit must be dropped/recreated manually and collation versions refreshed after PG 18 upgrade
